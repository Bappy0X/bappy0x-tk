Front End Languages

These languages are the pure fundamentals of any website and they comprise the front-end of any web application.

HTML5
Plain user interface.

HTML is the forefront of the front-end. It communicates with the browser, telling it what to display. It may also link to other sites for assets like Style Sheets and Scripts. HTML uses tags (known as elements) and DOM, which is very simplistic.
I am very advanced in this language and it is the first front-end language that I ever learnt.

Find out more!
CSS3
Styling for plain documents.

A CSS file is used in order to describe the presentation of a HTML document. This allows for complete control over how the website looks to the end user.
Along with HTML, I am very experienced in CSS and it was one of the first front-end languages that I learnt.

Find out more!
Bootstrap
User Interface Framework

Bootstrap uses more advanced CSS tricks in order to allow for appropriate scaling across all devices. It also has different presets and templates to allow for an easier and quicker experience for a developer. This site actually uses Bootstrap!
I have very vast experience with Bootstrap and I'm very fluent and confident in it.

Find out more!
JavaScript
Front-end code execution.

JavaScript is used for User Experience and controls the HTML document. It controls how elements act and respond to different types of input.
JavaScript is one of my favourite languages to use. It may not be high level but it is extremely flexible and can be used in many different ways.

Find out more!

Back End Languages

Back-end languages are the backbone of a web application, they will handle the processing and handling of web requests.

Python (Flask/Django)
Server-side code execution.

In order to create a dynamic web application using Python, you can use the Flask or Django packages. This site uses Flask, and it is my favourite for creating the back-end for web applications.
Python is my favourite language ever. It's extremely powerful and can do anything you'd like to throw at it. It can be used to create games, create discord bots, read your voice and even recognise your face!

Find out more!
Gunicorn
Client-Server Communication

Gunicorn is a WSGI HTTP Server. This, in laymans terms, essentially means that it handles the communication between client (The user, you) and the server (The back-end code).
Gunicorn is extremely easy to use on nearly any machine, and it's a great choice if you wouldn't like to complicate your WSGI.

Find out more!
JavaScript (Node.js & Express)
Hypertext Markup Language

Node.js executes JavaScript code outside of a browser. It is just as powerful as python when used correctly and can be used to do some very magical things. Along with the Express package, Node.js can become a web server to host a website.

Find out more!

Software Languages

These languages are used to create applications for a user to interact with.

JavaScript (Electron)
Build web-based applications.

Electron is built on Node.js and uses Chromium in order to function. It can use front end files (HTML, CSS, JS) or connect to a website to operate. Many applications are built on Electron, and it's very developer friendly. I personally use it in order to build desktop applications.

Find out more!
Dart
Hypertext Markup Language

Developed by Google, Dart is a programming language that is used to create applications on mobile platforms (Android and iOS) it can also be used in order to build desktop, server and web applications. It has C-Style syntax and is obejct oriented. It's able to compile native code or JavaScript.

Find out more!
Flutter
UI Framework

Flutter is a UI framework made for Dart. It can be used for applications on many platforms, including Android, iOS, Windows, Mac and Linux.

Find out more!
C# (.NET)
Build "Form" applications.

C# uses Microsoft's .NET library in order to create applications that are compatible with many different operating systems. It uses object oriented programming methods and is somewhat easy to learn - if you have prior experience with programming. C# is definitely one of my favourites for software development - it's really fun, and really intuitive.

Find out more!

Testing/CI+CD Services

When preparing for deployment of an application, it's important to check if it's functional. These applications help to do just that.

Trello
Project planning.

Trello is an intuitive planner for projects, companies and lots more. It's comprised of teams/organisations, boards, lists and cards. All of which are very customisable.
I use Trello in order to track bugs, issues, ideas and tasks for each of my projects.

Find out more!
TravisCI
External testing.

TravisCI is used to test applications to ensure that they function correctly. Testing is completely customisable and you can test many different forms of application. Regardless of your skillset, TravisCI is simplistic to use and can be learnt fairly easily. TravisCI is also free for ALL open source projects, and have other pricing available if your project isn't open source.

Find out more!
CircleCI
External testing.

CircleCI is a little more customisable than TravisCI. It's pricing is also a little cheaper and it has a range of different VM's to choose from.

Find out more!

Deployment Services

When launching applications to the web, I primarily use these services.

GitHub
Open Source file hosting.

GitHub is used in order to store all of my Git repositories online. It allows people to take a look at peoples projects and show/tell them how they can improve or expand on their work. Many projects are hosted entirely on GitHub and many developers are pro-open source.

My GitHub Find out more!
Heroku
PaaS

Heroku is a free platform as a service for website and application hosting. It's known for hosting Discord bots and many different websites. As it's free, it allows all programmers to host their presence online. However, completely free users have to link a form of payment in order to host their application 24/7.

Find out more!
Docker
PaaS

Docker is a Platform as a Service that allows for people to virtualise their applications. It has both free and premium tiers. It can also be run both on your own device, or in the cloud.

Find out more!
Glitch
PaaS

Glitch is a lot like Heroku, it allows users to host their website or application. Glitch applications can be hosted completely free of charge and even without an attached card. You are able to upgrade your application if your choose and the prices for these upgrades are extremely reasonable. Glitch applications automatically shut down if they're not in use, and will automatically start up if they recieve traffic. They also promote a helping, and you recieve more features, like custom domains, if you help people with their code - which I greatly approve of.

Find out more!

Data Stores

When data is required for an application, a data store must be used. Here is what I use.

PostgreSQL
Relational database.

PostgreSQL is a great relational database for SQL querying. It is also open source. Heroku also offers a free plan for PostgreSQL hosting.

Find out more!
Redis
Key-value database.

Key-value database that operates entirely in memory. It's great for queries that require super fast results - like caches.

Find out more!
SQLite
Serverless database.

A lightweight SQL database that is serverless and is self contained.
I mainly use this for testing and for smaller projects.

Find out more!
MongoDB
Non-SQL database.

A non-SQL and non-relational document oriented database. It looks and feels a lot like JSON.

Find out more!
